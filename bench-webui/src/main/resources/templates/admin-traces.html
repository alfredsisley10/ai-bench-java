AppMap trace coverage

With appmapMode != OFF, benchmarks ship runtime traces alongside the source code so the LLM can see the call graph the bug actually exercises. Real traces (under banking-app/<module>/tmp/appmap/junit/) are produced by running the module's JUnit suite under the AppMap Java agent — they reflect the live execution path, not a synthesized stub.

Without real traces, the harness falls back to synthetic JSON stubs that parse correctly but don't carry useful runtime information. Generating real traces is a one-time per-module cost (30s–5min depending on suite size); subsequent benchmarks pick up the cached .appmap.json files instantly.
