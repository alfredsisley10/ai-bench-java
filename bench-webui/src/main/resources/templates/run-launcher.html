Runtime options

Sits between the harness and the LLM and decides which code/trace excerpts get included in the prompt. Does not change which LLM is called — that's still the model selected above. Configure Navie on the AppMap setup page.

AppMap traces provide runtime context (call graphs, data flows, HTTP interactions) to the LLM solver. ON_RECOMMENDED is the default and balances trace quality with performance. ON_ALL gives maximum context but increases build time and token usage.

Number of independent solver attempts per bug. Multiple seeds measure consistency (pass@k). Each seed uses a different random seed for the LLM, producing varied solutions for the same prompt. Typical: 3 for quick runs, 5-10 for statistical significance.
