Gradle settings

Configuration for the Gradle subprocesses bench-webui spawns (banking-app boot, AppMap trace generation, mirror-resolve probes). Currently this page hosts the Artifactory mirror settings; future enhancements will surface gradle.properties validation and diagnostics here too. Independent from /proxy — you can configure either, both, or neither.

Configure Artifactory mirror

URL + credentials are forwarded to every Gradle subprocess the WebUI spawns; the corporate ~/.gradle/init.d/corp-repos.gradle.kts init script (written by scripts/build-health-check) also picks them up so manual ./gradlew calls inherit the same routing.

Heads up — many enterprises run multiple Artifactory mirrors. Tokens are scoped per mirror (e.g. artifactory-intg, artifactory-prod, artifactory-stage), and a token issued for one mirror returns 401 with "token failed verification" against another even when the URL works. Double-check the URL + token combination comes from the same Artifactory instance.
