Bug-definition leakiness lint

Static check over bugs/*.yaml for cases where the problem statement or hints accidentally name something that gives the fix away — the file or class the human edit touched, the production class derived from the hidden test, package path fragments, or the test method name itself. A leaky bug definition lets any model "solve" the bug from the description alone, with no code in the prompt; that result cannot be distinguished from pre-training contamination without the negative-control probe.
