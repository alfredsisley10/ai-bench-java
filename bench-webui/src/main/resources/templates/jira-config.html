Connection

Which one? Cloud is the Atlassian-hosted service at *.atlassian.net; only Cloud uses the email + API token combo. Server / Data Center is self-hosted; this WebUI defaults that case to Username + Password because most enterprise installs sit behind LDAP / SSO-mapped accounts and have that path enabled. PAT (Bearer auth, no email) is available too — it's the modern path on JIRA Datacenter 8.14+ and the supported alternative when admins have disabled basic auth.

Generate at Atlassian API tokens. Sent over the wire as Authorization: Basic base64(email:apiToken).

Create in JIRA → Profile → Personal Access Tokens (requires Server / Data Center 8.14+). Sent as Authorization: Bearer <pat> — no email or username is sent or required. If your JIRA is older than 8.14, use the Username + Password option instead.

For older JIRA Server / Data Center installations that haven't yet enabled PAT support. Sent as Authorization: Basic base64(user:pass).
