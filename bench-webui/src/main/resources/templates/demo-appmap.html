            How it flows: click 1 → wait for
            STARTING → RUNNING (banner above turns green). Then either
            click Open app for a browser session (a single-use
            in-memory token is exchanged for a cookie — no password
            prompt) or hit http://localhost:8080/api/...
            from curl. Click 2 to begin capturing, drive

            (it'll appear in the table below). Repeat 2 → 3 as
            many times as you like; only one recording can be active
            at a time. Click 4 when you're truly finished.
        

    


    

        
Record from tests

        

            Kicks off ./gradlew test in banking-app with
            ORG_GRADLE_PROJECT_appmap_enabled=true. The
            recording runs detached as a child JVM; the status panel
            above polls every 2 seconds with the live gradle log tail and
            updates the trace count as new AppMap files appear under
            banking-app/**/tmp/appmap/.
        

        

            Why a delay? Each test class' AppMap files
            are written when its test task completes, not as individual
            tests run. A full ./gradlew test across all
            generated modules can take 30–60s; narrow the scope with
            Module and Test filter for quicker
            iteration.

                        Test-filter syntax — examples and rules
                    
                    

                        Forwarded directly to Gradle as
                        --tests <value>; same syntax as
                        ./gradlew test --tests ….
                        
    
                            
  • PaymentLifecycleManagerTest —
                                run every test method in that class.
    • 
                            
  • com.omnibank.payments.AchCutoffTest
                                — fully qualified class name, useful when
                                the simple name is ambiguous.
    • 
                            
  • AchCutoffTest.testFinalWindow
                                — single test method.
    • 
                            
  • *.payments.*Test — wildcard:
                                every *Test class in any
                                .payments. package.
    • 
                            
  • *Repository* — every test
                                whose simple name contains "Repository".
    • 
                        

                        Quoting: on Windows
                        cmd, wrap patterns containing
                        * in double quotes
                        ("*.payments.*"). The WebUI passes
                        the value through verbatim so any quoting your
                        shell wants applies.

                        Multi-class: Gradle's
                        --tests only accepts one pattern
                        per invocation; to OR several classes,
                        scope by module above and let the
                        per-module test task pick them all up.
