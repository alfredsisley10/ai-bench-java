Interactive recording (drive the running app)

Start the banking app with the AppMap Java agent attached, then use Start / Stop here to bracket recordings while you exercise the running app from your browser. Stop captures the agent's trace and saves it under banking-app/app-bootstrap/tmp/appmap/interactive/ so it appears in the table below.

1b. Open app (auto-login)

Step 1b opens the banking app in a new tab and exchanges a single-use in-memory token for a session cookie — no password prompt. Then between steps 2 and 3, click around the banking app (or hit http://localhost:8080/api/... from curl) and the agent records every request that flows through the banking-app while the recording is active. Only one agent recording can run at a time.

Record from tests

Kicks off ./gradlew test in banking-app with ORG_GRADLE_PROJECT_appmap_enabled=true. The recording runs detached as a child JVM; the status panel above polls every 2 seconds with the live gradle log tail and updates the trace count as new AppMap files appear under banking-app/**/tmp/appmap/.

Why a delay? Each test class' AppMap files are written when its test task completes, not as individual tests run. A full ./gradlew test across all generated modules can take 30–60s; narrow the scope with Module and Test filter for quicker iteration.
