AppMap Setup

AppMap Navie is not an LLM — it's a context provider and interpretive layer that sits between the harness and whichever OpenAI-compatible endpoint it's pointed at. At benchmark time Navie does its own AppMap-driven retrieval, picks the most relevant code + trace excerpts, then forwards a tightly-focused prompt to the LLM. Configure the endpoint coordinates below; the LLM itself is whatever responds at that URL (typically the local Copilot bridge's OpenAI shim).

                    What it is. The Copilot bridge
                    (the VSCode extension this WebUI talks to) can
                    expose a small OpenAI-compatible HTTP
                    server on localhost. Tools that already speak the
                    OpenAI Chat Completions API — including
                    appmap navie — POST to that endpoint;
                    the bridge translates each call into a real GitHub
                    Copilot request using your VSCode session, then
                    streams the answer back in OpenAI wire format.
                    

                    When & why to enable it.
                    Required for the
                    AppMap Navie via Copilot path: Navie has
                    no Copilot client of its own, so without the shim
                    it can't use Copilot as its LLM. Skip it
                    if you've pointed Navie at some other
                    OpenAI-compatible endpoint instead (direct OpenAI,
                    Azure OpenAI, Ollama, etc.) — the shim only matters
                    when Copilot is the upstream model.
                    

                    How to enable. Open the VSCode
                    extension's webview and toggle
                    Enable OpenAI shim. Defaults to
                    127.0.0.1:11434 (loopback only, no
                    auth); change via
                    aiBench.copilotBridge.openAiPort /
                    bindAddress if those defaults clash with
                    something else on your machine.
