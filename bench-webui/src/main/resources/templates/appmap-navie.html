AppMap Navie — context provider setup

AppMap Navie is not an LLM — it's a context provider that sits between the harness and whichever OpenAI-compatible endpoint it's pointed at. At benchmark time Navie does its own AppMap-driven retrieval, picks the most relevant code + trace excerpts, then forwards a tightly-focused prompt to the LLM. Configure the endpoint coordinates below; the LLM itself is whatever responds at that URL (typically the local Copilot bridge's OpenAI shim).

Verify — real AppMap CLI round-trip

Spawns the AppMap CLI twice (same binary the harness invokes at benchmark time):

  1. appmap --version — proves the binary is present and executable on this host (fast, <2 s).
  2. appmap navie "@help …" with OPENAI_BASE_URL + OPENAI_API_KEY injected from the saved config — exercises the full CLI → bridge OpenAI shim → Copilot chain. The @help agent skips AppMap retrieval so the round-trip finishes quickly. (up to ~75 s)

This is the same code path the harness takes at benchmark time, so a green result here means the real AppMap Navie via Copilot benchmark provider will work.

Use it from a benchmark

Once the rows above are green, the Quick Benchmark dropdown surfaces AppMap Navie via Copilot automatically. Picking it tells the harness to invoke appmap navie against the endpoint configured here instead of calling the LLM directly — see the Live log on a Navie run for the actual routing path (look for [CONTEXT] Navie: … lines).
