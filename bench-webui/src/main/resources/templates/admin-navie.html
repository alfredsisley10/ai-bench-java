AppMap Navie precompute cache

The appmap-navie context provider in Layer-C reads from a per-bug cache populated by running the local appmap navie CLI ahead of time. Without a cached entry, the provider falls back to Oracle (the curated filesTouched list) and the audit page rationale notes the miss.

Each precompute fires a single Navie agentic run through the Copilot bridge (model copilot-gpt-4-1), which can take 5-30 minutes per bug. Jobs run one at a time (single-thread queue) since Navie monopolizes the bridge mutex anyway.
