Verify connectivity

Probes the endpoints Gradle needs to bootstrap a build — services.gradle.org (wrapper distribution), repo.maven.apache.org (Maven Central), api.foojay.io (JDK toolchain auto-provisioning), your configured Artifactory mirror, and the corporate proxy itself. Each probe runs through whatever proxy + auth + mirror settings you've saved above, so a green sweep here means the live config will work the next time you click Run gradlew --version on the demo page.

~5–15 seconds for the full sweep.

Detect from OS & VSCode

Reads proxy settings from the OS (macOS networksetup / Windows registry HKCU Internet Settings) and from VSCode user settings.json (http.proxy, http.proxyAuthorization, http.noProxy). Pre-fills the form below — you still have to click Save & apply to commit.

            HTTP clients (GitHub, JIRA, banking-app probes, AppMap agent

            -Dcom.sun.net.ssl.checkRevocation=false on the

Configure Artifactory mirror

        

            A Maven-compatible mirror used for plugin and dependency
            resolution instead of the public default repos. The URL +
            credentials are forwarded to every Gradle subprocess the
            WebUI spawns; the corporate
            ~/.gradle/init.d/corp-repos.gradle.kts init
            script (written by scripts/build-health-check)
            also picks them up so manual ./gradlew calls
            inherit the same routing.
        


        
        

            Heads up — many enterprises run multiple Artifactory mirrors.
            Tokens are scoped per mirror (e.g.
            artifactory-intg,
            artifactory-prod,
            artifactory-stage), and a token issued for
            one mirror returns 401 with
            "token failed verification" against another even
            when the URL works. Double-check the URL + token combination
            comes from the same Artifactory instance, not a
            sibling/staging one.
