GitHub Copilot (VSCode Bridge) — setup

The harness talks to Copilot through a local TCP endpoint (127.0.0.1 + an OS-assigned port) that a small VSCode extension exposes. The chosen port is written to a sidecar file at ~/.ai-bench-copilot.port for discovery. Walk through the three steps below; state icons turn green once each step succeeds.

  1. Step 1 — Install the VSCode extension

    The ai-bench Copilot Bridge VSCode extension binds a TCP listener on 127.0.0.1 and relays requests to GitHub Copilot through the VSCode Copilot extension. Pick whichever install link matches how your editor is reachable:

    Download copilot-bridge.vsix
    VSIX not built yet.

    The extension source lives in tools/copilot-bridge-extension/. Build it once and this page will auto-detect the artifact: 

    cd tools/copilot-bridge-extension
npm install
npm run package      # produces copilot-bridge.vsix

    Once downloaded, install it in VSCode via Extensions → … → Install from VSIX…, then run ai-bench: Start Copilot Bridge from the command palette. Override the bundled VSIX with the AI_BENCH_COPILOT_VSIX env var if you package it somewhere else.

    If your organization publishes the bridge to an internal marketplace (Open VSX, private registry, or a .vsix download server), replace the extension ID by setting the AI_BENCH_COPILOT_EXT_ID env var before launching bench-webui — the link above will pick it up on the next page load.

    Bridge endpoint:

  2. Step 2 — Start the bridge

    Open VSCode's command palette (⌘⇧P / Ctrl+Shift+P) and run ai-bench: Start Copilot Bridge. The extension binds a fresh TCP port on 127.0.0.1, writes the port to ~/.ai-bench-copilot.port, and the endpoint above flips to the live address. Override the sidecar path by setting the AI_BENCH_COPILOT_PORT_FILE env var before launching bench-webui.

  3. Step 3 — Verify & enumerate available models

    "List models" dials the bridge with {"op":"list-models"} and renders every Copilot-vended chat model the user is entitled to in the current VSCode session, including a synthetic auto entry that maps to whichever model GitHub returns first (its own "let GitHub pick" default).

    Adds each row below — and a synthetic auto entry — to the Registered models table. Each registered row's per-1K rates are auto-populated from the public model reference catalog below by id (e.g. copilot-gpt-4-1 picks up GPT-4.1's published $0.002/$0.008 rate). Copilot itself bills via seat, not per token — the rate shown is the underlying vendor's published rate, useful for chargeback estimates. Override per-model on the catalog if your org's contracted rate differs.
