The script checks out the break branch, runs failing tests, then checks out the fix branch and verifies tests pass.

Quick benchmark

Select an issue and configure the solver to run a full benchmark cycle (prepare → solve → score).

No LLM models registered yet. Register a model on the LLMs page — finish the GitHub Copilot (VSCode Bridge) or Corporate OpenAI Gateway wizard, then come back here. Quick benchmark requires at least one registered model to run.

Models come from Registered models on the LLMs page. Each option shows the model's display name + API identifier + per-1K prompt/completion cost.

A context provider sits between the harness and the LLM and decides what code/trace excerpts get included in the prompt. It does not change which LLM is called — that's still the model selected above.
None — the harness packs its own context (source files + AppMap traces per the AppMap mode below).
AppMap Navie — the AppMap CLI runs its retrieval engine over recorded AppMaps, picks the most relevant excerpts, and forwards a tightly-focused prompt through the bridge's OpenAI shim to the LLM selected above. Configure on the AppMap Navie page.

Controls whether the solver receives runtime context (call graphs, SQL, HTTP) recorded by the AppMap Java agent during the failing tests.
OFF — solver sees only source + test output. Fastest, lowest token cost; lowest fix rate.
ON_RECOMMENDED (default) — primes the solver with traces from the test classes the bug actually fails in. Best balance of context vs cost.
ON_ALL — primes the solver with every trace under banking-app/**/tmp/appmap/. Highest fix rate but ~3-5× the prompt-token cost and slower runs.

Number of independent solver attempts per bug — each run uses a different RNG seed for the LLM, producing different solutions for the same prompt. The harness measures pass@k: did at least one of the seeds attempts produce a passing fix? Typical values: 1 for spot-checking, 3 for normal runs (the default), 5–10 when you need statistical significance to compare two models. Cost scales linearly with seeds — a 5-seed run costs ~5× a 1-seed run.
