Testing with GitHub Copilot in VSCode

Step-by-step guide for running AI benchmark tests through the Copilot VSCode bridge.

1. Prerequisites

  • VSCode with GitHub Copilot extension installed and activated
  • Active GitHub Copilot subscription (individual or enterprise)
  • The ai-bench Copilot bridge extension installed in VSCode

2. Install the bridge extension

  1. Open VSCode
  2. Open the command palette (Cmd+Shift+P / Ctrl+Shift+P)
  3. Run Extensions: Install from VSIX...
  4. Select tooling/vscode-copilot-bridge/ from this project
  5. Or run: cd tooling/vscode-copilot-bridge && npm install && npm run compile

3. Activate the bridge

  1. Open the command palette
  2. Run AI Bench: Start Copilot Bridge
  3. The bridge listens on a Unix socket at:
  4. Status:

4. Run a benchmark with Copilot

  1. Go to Run benchmark
  2. Select copilot as the solver
  3. Choose your target bug and AppMap mode
  4. Launch the run — the harness will route prompts through the VSCode bridge

5. Reviewing Copilot logs for token usage

GitHub Copilot logs can be found in VSCode's Output panel:

  1. Open View > Output in VSCode
  2. Select GitHub Copilot from the dropdown
  3. Look for entries containing request.tokens and response.tokens
  4. The bridge extension also logs token counts to its own channel: AI Bench Bridge

Alternatively, the harness captures token counts from each Copilot response automatically. Check the run results page for per-request token breakdowns.

6. Comparing Copilot with other models

Run the same bug with different solvers to compare:

  • Copilot via the VSCode bridge
  • Corporate OpenAI via the Apigee gateway (configure in LLM settings)

The results page shows side-by-side metrics: pass rate, token usage, cost, patch complexity, and AppMap uplift.
